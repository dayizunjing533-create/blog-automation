- 性別: 男性

  - 朝起きられない
  - 何も続かない
  - 自信がない

## 1-5. 発信者のキャラクター設定
- 一人称: 「俺」または「自分」（「僕」は使わない）
- トーン: 友人に話すような自然体。敬語ベースだが堅くない。少し崩れた文体
- NG: 「〜しましょう！」の連発、過度な煽り、情報商材っぽい言い回し
- 吹き出し: 使うなら「過去の自分（ダメだった頃）」と「今の自分」の2パターンのみ。乱用しない

## 1-6. 記事構成ルール（全記事共通）

### リード文（300〜500文字）

以下の6ステップで構成する。本文を全て書き終えた最後にリード文を仕上げること。

ステップ1 読者の心に刺さる共感文
- 読者の悩みや現状を具体的に描写する
- 「自分のことを分かってくれている」と思わせる
- 単なる挨拶ではなく、リサーチで拾った「生の悩み」を使う
- 「当たり前」の共感は避ける（×「痩せたいと思っていませんか？」 ○「食べるつもりはないのに間食してしまう」）
- 抽象語ではなく頭に絵が浮かぶ具体語を使う（×「魚を食べましょう」 ○「朝食に鮭の切り身を焼いて食べる」）

ステップ2 問題提起（主張）
- 現状の問題や、その背景にある真の原因を指摘する
- 「実は〜」で書き始めると読者の興味を惹きやすい

ステップ3 権威性・実績
- 「誰が書いているか」を示し、記事の信頼性を担保する
- 資格や経験、実績を簡潔に伝える
- ない場合は、リサーチしたデータ（研究結果など）を根拠として提示する

ステップ4 記事内容の要約
- この記事に何が書かれているかを、魅力的な一文で伝える

ステップ5 読者のメリット（ベネフィット）
- 記事を読んだ後に読者がどんな「理想の状態」になれるかを明示する
- 「〜が分かる」「〜ができるようになる」の形で箇条書き可

ステップ6 最終的な結論（アンサーファースト）
- 記事の答えを出し惜しみせず真っ先に伝える
- 読者は忙しいため結論を先に知ることで安心して本文を読み進める

リード文の注意事項
- CTA（行動喚起）には触れない。目的はあくまで「本文を読ませること」
- 解決策の種をまく程度に留め、具体的な売り込みは最後に回す

### 見出し設計ルール

設計原則
- 見出しだけを上から順に読んで悩みから解決（CTA）まで一直線につながるように設計する（滑り台効果）
- 見出しを読むだけで結論（答え）がわかる文言にする（アンサーファースト）
- 1見出し1メッセージを徹底。情報を詰め込みすぎない
- 抽象から具体、大枠から詳細の順で並べる
- H2見出し: 3〜5個。キーワードまたは関連語を含める
- H3見出し: 各H2につき2〜4個
- H2→H3→H4の階層構造を厳守する

### 本文の構成ルール

#### 1. 本文の基本構成と展開の順序
人を動かす黄金の順序
- 記事全体および各セクションは「マインドセット（考え方・誤解の払拭）→ ノウハウ（やり方）→ ツール（道具）」の順で構成する
- この順序を崩さないこと

記事タイプ別の構成テンプレート
【ノウハウ記事の場合】
1. 定義: 何についての話か明確にする
2. 予備知識: ノウハウを理解するために必要な前提
3. 前提条件: 実践するために必要な環境や対象
4. メインノウハウ: ここを量・質ともに最大にする
5. その他: 補足・注意点・メリット・デメリット

【ノウハウ以外の記事（レビュー・主張など）の場合】
1. 主張: 結論を最初に述べる
2. 根拠: エビデンスを提示する
3. 具体例: 実体験や事例で裏付ける

見出し（H2）内のサブ構造
- 各H2の中身は以下の4要素で構成する
  1. WHY（根拠）: なぜそれが重要か
  2. WHAT（定義・中身）: 具体的に何を指すか
  3. HOW TO（実践手順）: どうやるか
  4. WHAT IF（事例・反論フォロー）: 実際どうなったか、反論への対応
- この4要素を入れることで論理の飛躍がなくなり説得力が増す
- 全てを必ず入れる必要はないが、WHYとHOW TOは必須

#### 2.「3つのNOT」対策

Web読者は「読まない・信じない・行動しない」という前提で執筆する。

【① 読まないの壁を突破する（視覚的工夫）】

アンサーファースト
- 結論（答え）を出し惜しみせず真っ先に伝える

スマホ最適化の見た目
- 一文は一呼吸で読める60〜80文字以内
- 一段落は150文字以内
- 3〜4行ごとに空白行を入れて「文字の壁」を回避

視覚的フックの設置
- 文章の羅列を避ける
- 箇条書き、表、吹き出し、太字、ボックス、画像を1画面に必ず1つは入れる

【② 信じないの壁を突破する（信頼性の担保）】

PREP法の徹底
- 結論(Point) → 理由(Reason) → 具体例(Example) → 結論(Point)の順で書く

生きた証拠（エビデンス）の提示
- ネットの情報をまとめるだけにしない
- 以下を必ず1つ以上含める
  - 自身の体験談（一次情報）
  - 写真や画像
  - 統計データや研究結果

先読みライティング
- 読者が読み進める中で抱く不安や反論を先回りしてフォローする
- 例:「本当か？」→ データで裏付け
- 例:「自分には無理だ」→ 最小ステップを提示

【③ 行動しないの壁を突破する（ベネフィットの提示）】

ベネフィットの明示
- 単なる機能や内容の説明ではなく、読者が手に入れられる「理想の未来」を具体的にイメージさせる
- 「〜ができるようになる」「〜な状態になれる」の形で書く

#### 3. 本文の具体的な表現ルール

具体語（「鮭」の原則）
- 「魚」ではなく「鮭」
- 「筋トレ」ではなく「スクワット1回」
- 読者の頭に具体的な絵が浮かぶ言葉を選ぶ

文末のリズム
- 同じ語尾（「〜です」「〜ます」など）の連続は2回まで
- 適宜体言止めを混ぜて文章にリズムを作る

不要な言葉を削る
- 「そして・また」などの接続詞は可能な限り削る
- 「これ・それ」などの指示代名詞は具体的な名称に置き換える
- 読み飛ばしても意味が通じる文章にする

中学生でもわかる平易な表現
- 難解な専門用語は避ける
- 使う場合は必ず解説を添える

読者への姿勢
- タイトルから最後まで滑り台のように流れる構成にする
- 読者に寄り添いながら書く
- 読者の利益・役立つ情報を優先する

本文の最終ゴール
- 読者の悩みを解決し、Googleに戻って再検索させない
- 顕在ニーズだけでなく潜在ニーズまで満たす
- この状態を記事内で完結させることが本文作成の最大の目的
