# /produce コマンド

このコマンドは「プロデューサー」として、記事制作の全工程（構成→本文→チェック→保存→SNS）を一括で進行する。

## 役割
/produce は以下の5つの社内部門をまとめて動かす指揮者。

| フェーズ | 担当部門 | やること |
|----------|----------|----------|
| PHASE 1 | 構成部門（/outline相当） | 構成案を生成し、人間の承認を取る |
| PHASE 2 | 執筆部門（/write相当） | 本文を生成し、人間の承認を取る |
| PHASE 3 | 品質管理部門（/check相当） | 全チェック項目を検証し、修正案を提示 |
| PHASE 4 | 出版部門 | ファイル保存・パイプライン更新 |
| PHASE 5 | SNS部門（/threads相当） | スレッズ投稿文を3パターン生成 |

---

## 前提条件（実行前に人間が確認すること）
- [ ] 執筆指示（Pattern A または Pattern B）が用意されている
- [ ] メインKWが人間によって確定・記入されている
- [ ] topics/experience.md の関連する体験ストック項目が記入済みである
- [ ] topics/pipeline.md のステータスが 🔄 IN_PROGRESS になっている

上記が揃っていない場合は、揃っていない項目を伝えて中断する。

---

## PHASE 1: 構成案生成

### 手順
1. CLAUDE.md を読み込む（文体ルール・KW運用・装飾ルールをすべて確認）
2. 執筆指示のメインKWを確認する。記載がなければ「メインKWを指定してください」と返して中断する
3. topics/experience.md を読み込み、記事に必要な体験ストックが揃っているか確認する
   - 空欄の項目がある場合: ⚠️「○○の体験が未記入です。空欄のまま進むと捏造リスクがあります。記入してから進めますか？」と確認する
4. 構成案を生成して出力する

### 出力フォーマット
```
🏭 PRODUCE | PHASE 1/5: 構成案生成
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

タイトル: （35文字以内、メインKWを前方に配置）

リード文: （50〜150文字、結論→メリット→要約の順）

H2-1: （共感・問題提起・経験提示。PREP法は適用しない）
  → この見出しで伝えること: 〜

H2-2: （PREP法）
  → この見出しで伝えること: 〜

H2-3: （PREP法）
  → この見出しで伝えること: 〜

H2-4:（必要な場合）
  → この見出しで伝えること: 〜

H2-5: （行動喚起・まとめ）
  → この見出しで伝えること: 〜

━━━━━━━━━━━━━━━━━━━━━━━━━━━━
✅ 承認する → PHASE 2（本文執筆）に進みます
🔄 修正する → 修正指示を入力してください
❌ 中断する → 作業を停止します
```

### ルール
- 構成案の承認なしでPHASE 2に進まない
- 人間が修正指示を出したら構成案を修正して再提示する
- 承認後、pipeline.md のメモ欄に「構成案承認済み」を記録する

---

## PHASE 2: 本文執筆

### 手順
1. 承認済みの構成案を確認する
2. CLAUDE.md の全ルールに従って本文を生成する（ですます調・PREP法・装飾ルール）
3. topics/experience.md から該当する体験を引用する
4. 体験ストックにない経験は書かない。書けない箇所は以下のように記載する:
   ```
   【要確認】○○の体験が体験ストックにありません。記入後に修正してください
   ```
5. 本文全文を出力する

### ルール
- 本文の承認なしでPHASE 3に進まない
- 「だ・である調」が1文字でも混入していたら自分で気づいて修正してから出力する
- AI臭い表現（「〜ではないでしょうか」「〜において」等）はCLAUDE.mdのリストで確認してから出力する

---

## PHASE 3: 品質チェック

### 手順
check.md のチェック項目に従ってセルフチェックを行い、結果を出力する。

### 出力フォーマット
```
🏭 PRODUCE | PHASE 3/5: 品質チェック
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. 文体（ですます調）: ✅ / ❌ →（❌なら該当箇所と修正案）
2. 一人称（「僕」メイン）: ✅ / ❌ →
3. KW（タイトル・H2・本文）: ✅ / ❌ →
4. 見出し構成（H2: 3〜5個・階層）: ✅ / ❌ →
5. PREP法（各H2）: ✅ / ❌ →
6. 根拠（各H2に数値・事例・体験1つ以上）: ✅ / ❌ →
7. noteで使えない装飾（テーブル等）: ✅ / ❌ →
8. リード文（結論→メリット→要約）: ✅ / ❌ →
9. 1文60字以内・段落150字以内: ✅ / ❌ →
10. NG表現（煽り・情報商材・上から目線）: ✅ / ❌ →
11. 体験ストック（引用した体験が実在するか）: ✅ / ❌ →
12. AI臭い表現（CLAUDE.mdの禁止リスト）: ✅ / ❌ →
13. 語尾3連続禁止: ✅ / ❌ →

━━━━━━━━━━━━━━━━━━━━━━━━━━━━
総合判定: ✅ 全クリア / ❌ X箇所に要修正

（❌がある場合）
修正案を適用しますか？
→「修正して」: 該当箇所のみ修正して再度出力
→「OKです」: そのままPHASE 4に進む
```

## PHASE 4: ファイル保存・パイプライン更新

### 手順
1. 確定した記事を `output/note/article-XXX.md` に保存する（XXXは3桁ゼロ埋め）
2. topics/pipeline.md の当該記事のステータスを ✅ REVIEW に更新する
3. 以下のメッセージを出力する:

```
🏭 PRODUCE | PHASE 4/5: ファイル保存
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

✅ output/note/article-XXX.md に保存しました
✅ pipeline.md のステータスを ✅ REVIEW に更新しました

noteへの実際の投稿は人間が行ってください。
投稿完了後、pipeline.md の公開日を記入して 🟢 PUBLISHED に更新してください。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━
次はPHASE 5（スレッズ投稿文）に進みますか？
→「はい」: スレッズ投稿文を3パターン生成します
→「後で」: ここで一旦終了します（後から /threads で生成できます）
```

## PHASE 5: スレッズ投稿文生成

### 手順
threads.md のルールに従って、完成記事からスレッズ投稿文を3パターン生成する。

### 出力フォーマット
```
🏭 PRODUCE | PHASE 5/5: スレッズ投稿文
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【パターン1: 常識否定型】
（フック1行 + 本文 + 問いかけ）

【パターン2: 実体験型】
（フック1行 + 本文 + 問いかけ）

【パターン3: 保存型】
（フック1行 + 本文 + 問いかけ）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━
使うパターンを選んでください（1 / 2 / 3）
選んだパターンをそのままコピペできる状態で最終出力します。

🎉 /produce 完了！お疲れさまでした。
```

---

## 中断・再開ルール
- 人間が「❌ 中断」と言った時点で作業を停止する
- 再開時は「PHASE Xから再開します」と言えば、そのフェーズから続きを進める
- 途中のPHASEは無駄にならない（例: PHASE 2承認済みならPHASE 3から再開できる）
