# /write コマンド

このコマンドは、承認済みの構成案をもとにnote記事の本文を生成する。

## 前提条件
- /outline で構成案が生成され、人間の承認を得ていること
- 承認なしで実行した場合は「先に /outline で構成案を承認してください」と返す

## 実行手順
1. CLAUDE.md を読み込む（文体ルール・PREP法・QUEST法・装飾ルール・体験ストックすべて参照）
2. 承認済みの構成案を確認する
3. H2セクションごとに本文を生成する
4. 各H2セクションはPREP法（Point→Reason→Example→Point）で書く
5. 全H2が完成したら、最後にリード文（50〜100文字）を生成する
6. 完成した記事全文を出力する

## 生成ルール
- CLAUDE.md の文体ルールを厳守（ですます調、一人称「僕」）
- 体験ストック（セクション11）に該当する経験があれば必ず引用する
- 体験ストックにない経験は捏造しない
- noteの装飾ルール（セクション10）に従う
- 画像が必要な箇所は【画像：○○の画面キャプチャ】と記載する
- コードブロックの前後に説明文を1行ずつ入れる
- 有料noteへのCTAは、人間が執筆指示で「CTA入れる」と指定した場合のみ記事末に1文入れる。指定がなければ入れない

## 出力形式
- noteにそのまま貼り付けられるMarkdown形式で出力する
- H2見出しの前に区切り線（---）を入れる
