# /plan コマンド

このコマンドは「編集長」として、次に書くべき記事を戦略的に判断し、執筆スケジュールを提案する。

## 役割
/plan は記事を書かない。「何を・いつ・なぜ書くか」だけを決める戦略コマンド。

## 実行手順
1. CLAUDE.md を読み込む（KW設計・ペルソナ・ブログの方向性を確認）
2. topics/pipeline.md を読み込む（記事ステータスを把握）
3. topics/keywords.md を読み込む（未記事化KWと優先度を確認）
4. output/note/ の記事ファイル一覧を確認する（公開済み記事の内容を把握）
5. 以下の優先ルールに従って「次に書くべき記事 TOP3」を選定する
6. 結果を以下のフォーマットで出力する
7. 人間の承認を待つ

## 優先ルール（上から順に適用）
1. **予告済みフォロー**: 公開済み記事の中で「次の記事で書きます」と言及している内容 → 最優先
2. **パイプライン計画済み**: pipeline.md の📝PLANNEDから優先度🔴の記事
3. **SEO核KW**: CLAUDE.mdのSEO核KW「Claude Code note 自動化」に近い記事
4. **体験ストック充足**: topics/experience.md の関連項目が記入済みの記事（捏造リスクが低い）
5. **ペルソナ直撃**: CLAUDE.mdのターゲットペルソナの悩みに最も刺さる記事

## ルール
- KWの変更・独自追加は禁止。人間が確認・承認したKWのみ使う
- 体験ストック（topics/experience.md）に未記入の項目がある場合は⚠️で明示する
- 出力後は人間の承認を待つ。承認なしで /outline や /produce には進まない
- 人間が「1位でいきます」と言ったら pipeline.md のステータスを 🔄 IN_PROGRESS に更新してから /produce を促す
